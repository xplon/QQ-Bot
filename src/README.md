# src

`src/` 目录存放 qq-bridge 的核心运行时代码。

当前主要文件：
- `index.js`：桥接服务的主干入口，包含状态维护、事件分发、策略判断与 LLM 交互逻辑。
- `mcp.js`：Model Context Protocol 工具链集成，定义与执行外部扩展（如 web_search 等）。

---

## 当前逻辑分布与职责

随着项目的演进，这一层已经不仅仅是消息转发，而是包含了完整的决策上下文。

### 1. 通讯与连接 (Transport)
- 与 NapCat/OneBot 建立稳定双向 WebSocket 连接（Event + API）。
- 监听并标准化 `message.group` 和 `message.private` 等事件。

### 2. 本地策略与过滤 (Policy & Filter)
- `checkAllow()` 函数强制执行黑白名单过滤，阻断受限请求。
- 群聊探测：被 @ 判定、群聊名字点名判定、疑问句探测以及群插话随机事件触发。

### 3. 持久化与状态记忆 (State Management)
- 将重要状态（`chatHistory`、`glossary`、群组近期记录、阻断名单）周期性保存至 `logs/state.json`。
- 在崩溃重启等情况下，通过 `loadPersistedState()` 实现记忆恢复。

### 4. 拟人行为决策 (Decision & Formatting)
- **等待时机**：根据时间与活跃状态，计算多梯度延迟。
- **消息积攒**：检测多条消息并发时积攒一并回复，防止逐条触发导致的“机器人刷屏”。
- **自然分段**：借助正则表达式按人类呼吸短点（标点符号界限）切分长篇大段文字。并剔除或转换 LLM 生成的 Markdown 及僵硬词汇。

### 5. 跨界能力延伸 (LLM & MCP)
- 调用外部聊天接口，将提取好的上下文注入大模型系统提示词中。
- 集成 `mcp.js`：让模型学会基于用户提问主动调取网页搜索功能，获取实时资讯并融入自身应答。

## 后续演进路线重构方案

经过初期的迭代验证，功能越来越丰富，但也导致 `index.js` 开始变得庞大。在朝着高级认知进化的目标下，建议进行如下拆分及升级：

### 重构模块化（可逐步实施）
- `transport/`：专门处理 NapCat 的连接、通信重组与 `WebSocket` 维护。
- `decision/`：承接 `checkAllow`、`computeDelayMs`、忙碌状态判定等行为分析模块。
- `memory/`：抽象独立的记忆处理中心，目前基于 state.json 的近期记忆，未来演进至长期记忆管理模块。
- `llm/`：调用、封装与大模型的对接，包括 Prompt 构造与工具编排。

### 新增核心展望特性支持
1. **用户画像系统**：设计 `UserProfile` 类，在用户交互时主动提取特点并持久化。
2. **多线程话题跟踪分析**：在 Decision 层或 Memory 层增加对话线编排，实现同一群内不同人群并行话题的回溯。
3. **情绪与语义分析中心**：在进到回复流程前，通过本地小型分析器或 LLM 快速判断语境氛围，实现情感情绪检测。
4. **多模态与内容感知接入**：通过 NapCat 的富媒体协议解析，接收图片/表情包记录，增加 OCR 或多模态模型的预处理模块。

这些新特性将让机器人从“反应敏捷的答录机”蜕变为“拥有情绪和多话题感知能力的智能玩伴”。
