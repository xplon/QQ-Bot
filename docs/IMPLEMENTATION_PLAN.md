# qq-bridge 项目执行计划

本计划反映了 `qq-bridge` 基于当前已实现成果（双向通讯、拟人策略、MCP 工具调用与状态持久化）迈向高阶智能会话体的新阶段蓝图。

---

## 当前已达成（Phase 1 & 2）

- [x] NapCatQQ (OneBot) 消息双向通道建设（WebSocket）。
- [x] 私聊、群聊的基础触发及风控拦截策略（白名单机制）。
- [x] 行为层拟人化实现：
  - 基于作息与不同场景计算的动态延时模型。
  - 自然分句与分发节流。
- [x] MCP (Model Context Protocol) 框架实测，实现 Web Search 功能。
- [x] 近期对话记忆缓存与 `state.json` 的本地方案持久化。

---

## 中期规划（Phase 3：认知与记忆升级）

本阶段重点在于让机器人具备长期视角和情绪认知的“高拟真灵魂”。

### 1. 持久化私聊、群聊用户的画像
- **目标**：能识别并刻画长期互动的用户特征。
- **实施计划**：
  1. 架构调整：引入本地轻量级数据库（如 SQLite 或 JSON 碎片化管理）。
  2. 画像抽取：定义一套 UserProfile Model，包含：口语习惯、常见称呼、交流偏好频率、历史重要事件（生日、喜好等）。
  3. LLM 集成：定期利用 LLM 通过背景进程或定期总结机制对上下文进行二次分析，提取特征写入画像库。

### 2. 实现长期记忆系统
- **目标**：跳出单次上下文滑动窗口（近期记忆），让聊天具备长时连续性。
- **实施计划**：
  1. 将单纯的 `history` 列表转化为“向量数据库的嵌入（Embedding）索引”架构。
  2. 实现检索逻辑（RAG：Retrieve-Augmented Generation）：在每次生成 Prompt 时，除了带上最近的对话，同步搜索长期记忆库相关的历史提及物。
  3. 保留记忆衰减与遗忘机制体系。

### 3. 检测当前对象 / 群内情绪状态
- **目标**：让机器人表现得“有眼力见儿”。
- **实施计划**：
  1. 引入专门的的情绪极性分析模型，或者使用 LLM 补充额外一次 prompt 用于标注当前语气的强烈度。
  2. 结合情绪打分，调整 Decision 层的动作参数：如愤怒情绪下增加沉默概率；悲伤情绪下切换温声抚慰类回复。

### 4. 厘清消息的对话线，区分并行话题
- **目标**：改善群内多人多话题同时发生时的“上下文胡乱跨越合并”乱象。
- **实施计划**：
  1. 利用消息的引用 (`replyToId`) 特性做主要溯源。
  2. 加入语义相似度聚合：将同时间段群组的不同发言切片归宿到“Topic A”、“Topic B”、“Topic C”。
  3. 回复定位：当决定回应特定信息时，将注入对应线索所在的独立 Context。

## 长期规划（Phase 4：多模态进阶）

### 5. 接入表情包和图片识别
- **目标**：看懂网络热梗与视觉图像。
- **实施计划**：
  1. 拦截 NapCat 中的 `image`、`face` 等 CQ 码 /富媒体消息，提前解析 url。
  2. 搭建 / 接入多模态视觉模型（Vision Model），描述出图片关键信息或提取梗图上的文字。
  3. 模型反向输出动作标签：允许回复内置好的图片映射内容（比如触发“发送[猫猫头]表情包”函数）。
- 私聊发“你好” → OpenClaw 可收到并回消息
- 白名单群内 @ → OpenClaw 可收到并回消息
- 非白名单群 → 不回
- 超长回复 → 自动拆成 2~4 条短句
- 回复不带 markdown

### 阶段 2 验收
- 夜间回复变慢或不回
- 连续多条消息会等一下再回
- 偶尔出现“稍后回复”风格
- 支持简单主动问候

## 注意事项

- 个人号接入没有绝对安全
- 第一版必须保守
- 主动发消息功能需要额外策略，不应默认打开
- 所有群聊默认视为高风险环境
